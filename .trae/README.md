# Artefactos de planificación

Los archivos de `.trae/` son planes, especificaciones y notas de trabajo del
repositorio. No representan necesariamente la API o el modelo vigente.

Para conocer el estado actual, usar este orden:

1. código publicado y `package.json`;
2. documentación vigente en `src/docs/core/` y `src/docs/<dominio>/`;
3. migraciones y RFCs para contexto histórico;
4. documentos de este directorio como registro del trabajo realizado o pendiente.

Los nombres y rutas que aparecen en planes antiguos pueden referirse a archivos ya
eliminados. No deben utilizarse como imports ni como instrucciones de integración.
